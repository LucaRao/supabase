Supabase市场汇集了你需要的所有工具，以扩展你的Supabase项目。这包括。

- [专家](https://supabase.com/partners/experts) - 合作伙伴，帮助你建立和支持你的Supabase项目。
- [集成](https://supabase.com/partners/integrations) - 用外部授权、缓存、托管和低代码工具扩展你的项目。

## 构建一个集成

Supabase提供了几个集成点。

- Postgres连接](/docs/guides/database/connecting-to-postgres)。任何能与Postgres一起使用的东西也能与Supabase项目一起使用。
- 项目REST API](/docs/guides/api#rest-api) 和客户端库。
- Project GraphQL API](/docs/guides/api#graphql-api)。
- 平台API](/docs/reference/api)。

##列出你的集成

[申请加入合作伙伴计划](https://supabase.com/partners/integrations#become-a-partner)，在合作伙伴市场和Supabase文档中列出您的集成。

评估整合的标准如下。

- **商业可行性**
  虽然我们欢迎每个人建立一个集成，但我们只列出那些被认为是长期可行的公司。这包括正式的商业注册和银行账户，有意义的收入，或风险投资的支持。我们要求这个标准以确保市场的健康。
- **合规性**
  集成项目不应侵犯Supabase的品牌/商标。简而言之，你不能在名称中使用 "Supabase"。由于列表出现在Supabase域名上，我们不希望误导开发人员，使其认为集成是一个官方产品。
- **服务水平协议**
  所有的列表都需要有自己的条款和条件、隐私政策和可接受的使用政策，公司必须有资源来满足他们的服务水平协议。
- **可维护性**
  所有的整合都需要Supabase的维护和功能，公司可能会被评估你在很长一段时间内保持功能的能力。
